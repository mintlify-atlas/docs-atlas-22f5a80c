## Query subscribers

<CodeGroup>
```bash GET /api/subscribers
curl -u 'username:password' 'https://listmonk.mysite.com/api/subscribers' \
  -G \
  -d 'page=1' \
  -d 'per_page=100' \
  -d 'list_id=1' \
  -d 'subscription_status=confirmed'
```
</CodeGroup>

Retrieves a paginated list of subscribers with optional filtering.

**Authentication:** Required  
**Permission:** `subscribers:get_all` or `subscribers:get`

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of results per page (max 100)
</ParamField>

<ParamField query="query" type="string">
  SQL expression for advanced filtering (requires `subscribers:sql_query` permission)
</ParamField>

<ParamField query="search" type="string">
  Search term to filter subscribers by name or email
</ParamField>

<ParamField query="list_id" type="integer">
  Filter by list ID (can be specified multiple times)
</ParamField>

<ParamField query="subscription_status" type="string">
  Filter by subscription status: `unconfirmed`, `confirmed`, or `unsubscribed`
</ParamField>

<ParamField query="order_by" type="string" default="id">
  Field to order by: `id`, `email`, `name`, `created_at`, `updated_at`
</ParamField>

<ParamField query="order" type="string" default="asc">
  Sort order: `asc` or `desc`
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="results" type="array">
      Array of subscriber objects
      <Expandable title="subscriber object">
        <ResponseField name="id" type="integer">
          Subscriber ID
        </ResponseField>
        <ResponseField name="uuid" type="string">
          Unique UUID for the subscriber
        </ResponseField>
        <ResponseField name="email" type="string">
          Email address
        </ResponseField>
        <ResponseField name="name" type="string">
          Subscriber name
        </ResponseField>
        <ResponseField name="status" type="string">
          Status: `enabled`, `disabled`, or `blocklisted`
        </ResponseField>
        <ResponseField name="attribs" type="object">
          Custom attributes as key-value pairs
        </ResponseField>
        <ResponseField name="lists" type="array">
          Array of list subscriptions
        </ResponseField>
        <ResponseField name="created_at" type="string">
          Creation timestamp
        </ResponseField>
        <ResponseField name="updated_at" type="string">
          Last update timestamp
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="query" type="string">
      The SQL query used for filtering
    </ResponseField>
    <ResponseField name="total" type="integer">
      Total number of subscribers matching the query
    </ResponseField>
    <ResponseField name="per_page" type="integer">
      Results per page
    </ResponseField>
    <ResponseField name="page" type="integer">
      Current page number
    </ResponseField>
  </Expandable>
</ResponseField>

## Create subscriber

<CodeGroup>
```bash POST /api/subscribers
curl -u 'username:password' 'https://listmonk.mysite.com/api/subscribers' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "john@example.com",
    "name": "John Doe",
    "status": "enabled",
    "lists": [1, 2],
    "attribs": {
      "city": "Bangalore",
      "projects": 3
    },
    "preconfirm_subscriptions": true
  }'
```
</CodeGroup>

Creates a new subscriber.

**Authentication:** Required  
**Permission:** `subscribers:manage`

### Request Body

<ParamField body="email" type="string" required>
  Email address (must be valid and unique)
</ParamField>

<ParamField body="name" type="string" required>
  Subscriber name (max 2000 characters)
</ParamField>

<ParamField body="status" type="string" default="enabled">
  Subscriber status: `enabled`, `disabled`, or `blocklisted`
</ParamField>

<ParamField body="lists" type="array">
  Array of list IDs to subscribe to
</ParamField>

<ParamField body="attribs" type="object">
  Custom attributes as key-value pairs (JSON object)
</ParamField>

<ParamField body="preconfirm_subscriptions" type="boolean" default="false">
  If true, subscriptions are marked as confirmed without sending opt-in email
</ParamField>

### Response

Returns the created subscriber object.

## Delete subscribers by query

<CodeGroup>
```bash POST /api/subscribers/query/delete
curl -u 'username:password' -X POST 'https://listmonk.mysite.com/api/subscribers/query/delete' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "subscribers.attribs->>\"city\" = \"Bangalore\"",
    "list_ids": [1, 2],
    "subscription_status": "unsubscribed"
  }'
```
</CodeGroup>

Deletes subscribers based on an arbitrary SQL query.

**Authentication:** Required  
**Permission:** `subscribers:manage` and `subscribers:sql_query`

### Request Body

<ParamField body="query" type="string">
  SQL expression for filtering (requires `subscribers:sql_query` permission)
</ParamField>

<ParamField body="search" type="string">
  Search term to filter subscribers
</ParamField>

<ParamField body="list_ids" type="array">
  Filter by list IDs
</ParamField>

<ParamField body="subscription_status" type="string">
  Filter by subscription status
</ParamField>

<ParamField body="all" type="boolean">
  If true, deletes all subscribers (ignores other filters)
</ParamField>
