## Query campaigns

<CodeGroup>
```bash GET /api/campaigns
curl -u 'username:password' 'https://listmonk.mysite.com/api/campaigns' \
  -G \
  -d 'page=1' \
  -d 'status=running' \
  -d 'query=newsletter'
```
</CodeGroup>

Retrieves a paginated list of campaigns.

**Authentication:** Required  
**Permission:** `campaigns:get_all` or `campaigns:get`

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Results per page
</ParamField>

<ParamField query="query" type="string">
  Search term to filter campaigns by name
</ParamField>

<ParamField query="status" type="string">
  Filter by status (can be specified multiple times): `draft`, `scheduled`, `running`, `paused`, `finished`, `cancelled`
</ParamField>

<ParamField query="tag" type="string">
  Filter by tag (can be specified multiple times)
</ParamField>

<ParamField query="order_by" type="string" default="created_at">
  Field to order by
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order: `asc` or `desc`
</ParamField>

<ParamField query="no_body" type="boolean" default="false">
  If true, excludes campaign body from response
</ParamField>

## Create campaign

<CodeGroup>
```bash POST /api/campaigns
curl -u 'username:password' 'https://listmonk.mysite.com/api/campaigns' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Monthly Newsletter",
    "subject": "Newsletter for {{ .Subscriber.FirstName }}",
    "lists": [1, 2],
    "type": "regular",
    "content_type": "richtext",
    "body": "<p>Hello {{ .Subscriber.FirstName }}!</p>",
    "altbody": "Hello!",
    "from_email": "newsletter@example.com",
    "messenger": "email",
    "template_id": 1,
    "tags": ["newsletter"],
    "send_at": "2024-12-01T10:00:00Z"
  }'
```
</CodeGroup>

Creates a new campaign in draft status.

**Authentication:** Required  
**Permission:** `campaigns:manage_all` or `campaigns:manage`

### Request Body

<ParamField body="name" type="string" required>
  Campaign name (max 2000 characters)
</ParamField>

<ParamField body="subject" type="string" required>
  Email subject line (supports template variables)
</ParamField>

<ParamField body="lists" type="array" required>
  Array of list IDs to send to
</ParamField>

<ParamField body="body" type="string" required>
  Campaign body content
</ParamField>

<ParamField body="type" type="string" default="regular">
  Campaign type: `regular` or `optin`
</ParamField>

<ParamField body="content_type" type="string" default="richtext">
  Content type: `richtext`, `html`, `markdown`, `plain`, `visual`
</ParamField>

<ParamField body="from_email" type="string">
  From email address (defaults to system setting)
</ParamField>

<ParamField body="altbody" type="string">
  Plain text alternative body
</ParamField>

<ParamField body="send_at" type="string">
  ISO 8601 timestamp for scheduled sending (null for immediate)
</ParamField>

<ParamField body="messenger" type="string" default="email">
  Messenger backend to use
</ParamField>

<ParamField body="template_id" type="integer">
  Template ID to use
</ParamField>

<ParamField body="tags" type="array">
  Array of tags
</ParamField>

<ParamField body="headers" type="array">
  Custom email headers (array of objects with key-value pairs)
</ParamField>

<ParamField body="media" type="array">
  Array of media IDs to attach
</ParamField>

## Update campaign

<CodeGroup>
```bash PUT /api/campaigns/:id
curl -u 'username:password' -X PUT 'https://listmonk.mysite.com/api/campaigns/1' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Updated Newsletter",
    "subject": "New Subject",
    "body": "<p>Updated content</p>"
  }'
```
</CodeGroup>

Updates a campaign. Only campaigns in `draft`, `scheduled`, or `paused` status can be updated.

**Authentication:** Required  
**Permission:** `campaigns:manage_all` or `campaigns:manage`

### Path Parameters

<ParamField path="id" type="integer" required>
  Campaign ID
</ParamField>

### Request Body

Same fields as create campaign. Only provided fields will be updated.

## Update campaign status

<CodeGroup>
```bash PUT /api/campaigns/:id/status
curl -u 'username:password' -X PUT 'https://listmonk.mysite.com/api/campaigns/1/status' \
  -H 'Content-Type: application/json' \
  -d '{"status": "running"}'
```
</CodeGroup>

Changes a campaign's status to start, pause, or cancel it.

**Authentication:** Required  
**Permission:** `campaigns:manage_all` or `campaigns:manage`

### Path Parameters

<ParamField path="id" type="integer" required>
  Campaign ID
</ParamField>

### Request Body

<ParamField body="status" type="string" required>
  New status:
  - `running`: Start sending the campaign
  - `paused`: Pause a running campaign
  - `scheduled`: Schedule a draft campaign
  - `cancelled`: Cancel a campaign
</ParamField>

## Get campaign analytics

<CodeGroup>
```bash GET /api/campaigns/analytics/:type
curl -u 'username:password' 'https://listmonk.mysite.com/api/campaigns/analytics/views' \
  -G \
  -d 'id=1' \
  -d 'id=2' \
  -d 'from=2024-01-01T00:00:00Z' \
  -d 'to=2024-12-31T23:59:59Z'
```
</CodeGroup>

Retrieves campaign analytics data.

**Authentication:** Required  
**Permission:** `campaigns:get_analytics`

### Path Parameters

<ParamField path="type" type="string" required>
  Analytics type: `views`, `clicks`, or `links`
</ParamField>

### Query Parameters

<ParamField query="id" type="integer" required>
  Campaign ID (can be specified multiple times)
</ParamField>

<ParamField query="from" type="string" required>
  Start date (ISO 8601)
</ParamField>

<ParamField query="to" type="string" required>
  End date (ISO 8601)
</ParamField>

### Response

Returns analytics data with timestamps and counts for the specified period.
